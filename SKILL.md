# 경기도의회 의원활동 언론보도 리포트

경기도의회 보도자료 페이지(`ggc.go.kr/site/main/xb/lwmkr/lawmakerpressrelease`)에서 **오늘자** 의원 보도자료를 수집하고, 각 의원명으로 네이버 뉴스를 검색해 **외부 언론 기사**를 매칭한 뒤, 상임위별로 묶어 정해진 포맷의 텍스트를 만든다. 마지막에 사용자 승인을 받아 **카카오톡 '나에게 보내기'**로 전송한다.

## 동작 방식 (요약)

1. ggc 보도자료 목록(정적 HTML)을 내장 `fetch`로 페이지별로 훑어 오늘 날짜 항목만 수집한다. (목록은 최신순이라 오늘 이전 날짜가 나오면 중단)
2. 각 상세 페이지 본문에서 `의원명 / 상임위 / 직책 / 정당`을 파싱한다.
3. 의원명 + "경기도의회"로 네이버 뉴스(프록시, 키 불필요)를 최신순 검색해 가장 적합한 기사(제목·언론사·URL)를 고른다.
4. 상임위별로 그룹화해 아래 포맷의 텍스트와 `report_YYYY-MM-DD.txt`/`.json` 파일을 만든다.
5. 미리보기를 사용자에게 보여주고, 승인 시 **카카오톡 '나에게 보내기'(나챗방)로 전송한다. 이 전송이 스킬의 마지막 필수 단계다.**

## 출력 포맷 (고정)

```
〔경기도의회 의원활동 언론보도〕

<상임위 짧은이름>

○ 의원명 직책, 기사제목(언론사)
▶ 기사 URL
```

- 정당 필터를 쓰면 헤더가 `〔경기도의회 더불어민주당 의원활동 언론보도〕`처럼 바뀐다.
- 외부 언론 기사를 못 찾은 항목은 ggc 보도자료 제목/링크로 대체하고 언론사는 `경기도의회`로 표기한다.

## 1. 리포트 생성 (스크립트 실행)

스킬 디렉터리의 `build_report.js`를 실행한다. 출력 파일은 `GGC_OUT`(미지정 시 현재 디렉터리)에 저장된다.

```bash
# 기본: 오늘 / 전체 의원
GGC_OUT="$(pwd)" node ~/.claude/skills/ggc-press-report/build_report.js

# 특정 날짜 / 특정 정당만 (예: 2026-06-16, 더불어민주당)
GGC_OUT="$(pwd)" node ~/.claude/skills/ggc-press-report/build_report.js 2026-06-16 더불어민주당
```

인자:
- 1번째: 날짜 `YYYY-MM-DD` (생략 시 오늘)
- 2번째: 정당 (`더불어민주당`/`국민의힘` 등, 생략 시 전체)

Windows PowerShell에서 실행할 때는 환경변수를 분리해 지정한다.
```powershell
$env:GGC_OUT = (Get-Location).Path
node "$env:USERPROFILE\.claude\skills\ggc-press-report\build_report.js"
```

> 외부 패키지가 필요 없다. Node 18+의 내장 `fetch`만 사용하므로 별도 설치 없이 어느 폴더에서나 실행된다.

## 2. 결과 확인

- 스크립트 stdout의 `===== REPORT_BEGIN =====` ~ `===== REPORT_END =====` 사이가 전송할 본문이다.
- `[INFO] TXT_PATH=...` 줄에 저장된 txt 절대경로가 나온다. 이 파일을 Read로 읽어 정확한 본문을 확보한다.
- 수집 0건이면 "오늘 등록된 의원 보도자료가 없습니다." 로 나온다 — 이 경우 사용자에게 알리고 전송 여부를 묻는다.

## 3. 미리보기 → 승인 → 카카오 '나에게 보내기' 전송 (필수 마지막 단계)

**이 스킬은 리포트 생성으로 끝나지 않는다. 카카오톡 '나에게 보내기'(나챗방) 전송까지 완료해야 작업이 끝난 것이다.**

**중요: 전송은 사용자 권한이 필요한 액션이다. 반드시 미리보기 후 승인받는다.**

1. `report_YYYY-MM-DD.txt`를 Read로 읽어 정확한 본문을 확보하고, 본문 전체를 사용자에게 미리보기로 보여준다.
2. 사용자가 "전송"/"보내"/"yes" 등 명시적으로 답한 경우에만 `mcp__27b8548f-38a6-48fd-b630-3d7f54c6ef41__KakaotalkChat-MemoChat` 도구로 본문 텍스트를 전송한다. (이 도구가 deferred 상태면 `ToolSearch`로 먼저 로드한다.)
3. 전송 결과(성공/실패)를 보고한다.

> **긴 본문 처리**: MemoChat 스키마상 권장 길이는 200자지만 그 이상도 전송된다. 만약 길이 제한으로 전송이 거부되면, 헤더 `〔…〕`를 첫 메시지에 두고 **상임위(`<…>`) 블록 단위로 나누어 순차 전송**한다. 한 의원 항목(○ + ▶ 두 줄)이 쪼개지지 않도록 블록 경계에서만 분할한다.

## 매핑 보강 (선택)

- 언론사명은 `build_report.js`의 `OUTLET`(도메인→언론사) 맵으로 결정한다. 새 언론사가 도메인으로만 표시되면 이 맵에 추가한다.
- 상임위 짧은이름은 `SHORT` 맵으로 결정한다. 새 위원회는 여기에 추가한다.

## 알려진 한계

- 네이버 검색은 "의원명 최신 기사"를 고르므로, 그날 보도자료와 다른 주제의 기사가 매칭될 수 있다(특히 동명이인·선거 기사). 의심되면 본문을 확인하고 사용자에게 알린다.
- 의원 정보가 본문에 `이름 직책(정당, 지역구)` 형태로 없으면 파싱에서 제외된다(위원회 명의 보도자료 등). 이는 의도된 동작이다.

## 매일 자동화

이 스킬은 카카오 로컬 MCP를 사용하므로 **이 PC에서 실행**해야 자동 전송이 된다. 매일 정해진 시각 자동 실행이 필요하면 `/loop` 또는 Windows 작업 스케줄러로 이 스킬(또는 `build_report.js` + 카카오 전송)을 호출하도록 안내한다. 원격(클라우드) 루틴은 로컬 카카오 MCP를 호출할 수 없어 전송 단계가 불가능하다.
